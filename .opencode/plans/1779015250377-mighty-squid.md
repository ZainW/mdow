# Replace Comark HTML With Comark React

## Goal

Replace the desktop renderer's `@comark/html` pipeline with `@comark/react`, remove markdown `dangerouslySetInnerHTML`, and preserve the current user-facing behavior: headings/outline, scroll spy, copy-code buttons, Mermaid diagrams, syntax highlighting, and in-document search.

## Key Findings

- Current HTML renderer is isolated to `apps/desktop/src/renderer/src/lib/markdown.ts`.
- `@comark/react` is already installed and exports `ComarkRenderer`, which renders an existing `ComarkTree` synchronously in the client.
- `ComarkRenderer` accepts tag/component overrides via `components`, including `h1`, `pre`, and custom tags such as `mermaid`.
- The current implementation post-processes an HTML string for heading IDs, heading extraction, code block wrappers, copy buttons, and Mermaid placeholders.
- `useDocumentSearch` is coupled to HTML strings and must become a DOM enhancement hook after React rendering.

## Implementation Plan

1. Update `markdown.ts` to stop importing `renderHTML` from `@comark/html`.

2. Import Comark tree/node types from `comark` and change `RenderResult` to return the parsed tree instead of HTML:

   ```ts
   export interface RenderResult {
     tree: ComarkTree
     mermaidBlocks: { id: string; code: string }[]
     headings: DocHeading[]
   }
   ```

3. Keep `initMarkdown()` and `renderMarkdown(text)` as the public API. `renderMarkdown` should still call the existing parser with `highlight`, `mermaid`, and `breaksOutsideCode` plugins.

4. Add a small tree traversal in `markdown.ts` that mutates/annotates the parsed tree before returning it:

   - For `h1` through `h4`, extract plain text from descendants.
   - Generate IDs using the existing `slugifyHeading` algorithm and duplicate suffix behavior.
   - Set `node[1].id` directly on the Comark heading node so React renders stable native heading IDs.
   - Push `{ level, text, id }` into `headings`.
   - For `mermaid` nodes, read `attrs.content`, assign a generated `id` when missing, set `attrs.id`, set a class such as `mermaid mermaid-container`, and push `{ id, code }` into `mermaidBlocks`.

5. Remove `escapeAttr` and all DOM wrapper post-processing from `markdown.ts`.

6. Update `MarkdownView.tsx` state initialization and empty states from `{ html: '' }` to an empty parsed result shape. Prefer `RenderResult | null` or a small local `emptyRenderResult` if that keeps effects simpler.

7. Import `ComarkRenderer` from `@comark/react` and render the parsed tree inside the existing `.markdown-body` container:

   ```tsx
   <div ref={contentRef} className="... markdown-body ..." style={...}>
     {renderResult ? (
       <ComarkRenderer tree={renderResult.tree} components={markdownComponents} />
     ) : null}
   </div>
   ```

8. Add React component overrides in `MarkdownView.tsx` or a small `MarkdownRenderer.tsx`:

   - `pre`: render `<div className="code-block-wrapper relative">`, the original `<pre>`, and a React-managed `.copy-code-btn`.
   - `mermaid`: render an empty mount element, e.g. `<div id={id} className="mermaid mermaid-container" />`, and do not render the raw diagram text.

9. Replace the delegated copy-code click effect in `MarkdownView.tsx` with local React state in the `pre` override:

   - Use a ref to the `<pre>` or wrapper.
   - Copy `textContent` so highlighted nested spans copy as plain text.
   - Preserve current classes, `aria-label`, `title`, and `data-copied='true'` behavior so existing CSS continues to work.

10. Update `useDocumentSearch.ts` to operate on the live rendered DOM instead of HTML strings:

   - New API: `useDocumentSearch(containerRef, query, renderKey)` where `renderKey` can be `renderResult` or a stable version counter.
   - Remove existing search marks before applying new marks.
   - Walk text nodes under `containerRef.current`, excluding existing `mark.search-highlight` nodes.
   - Wrap matches in `<mark class="search-highlight" data-match-index="...">`.
   - Keep `matchCount`, `currentIndex`, `next`, `prev`, and `clear` return values.
   - Keep the effect that adds `.search-highlight-active` and scrolls the active match into view.

11. Update `MarkdownView.tsx` search usage:

   - Stop reading `highlightedHtml`.
   - Pass `contentRef`, the active query, and the current render result/version to `useDocumentSearch`.
   - Keep `SearchBar` props and close behavior unchanged.

12. Update Mermaid rendering effects:

   - Key the Mermaid effect off `renderResult?.mermaidBlocks` or a render version instead of `html`.
   - Keep `renderMermaidBlocks(renderResult.mermaidBlocks)` after React has mounted placeholders.
   - In `lib/mermaid.ts`, clear the mount element before rendering and preserve `mermaid mermaid-container` on success; on failure, set `mermaid-error` and text as today.

13. Update scroll-spy dependencies in `MarkdownView.tsx` from `html` to the current render result/version. The existing `querySelectorAll('h1[id], h2[id], h3[id], h4[id]')` can remain because IDs are now rendered by React.

14. Remove the `@comark/html` dependency from `apps/desktop/package.json` and update `pnpm-lock.yaml` via package manager install after implementation.

## Important Details

- Do not use `Comark`; it is typed as async and is less suitable for this client-side renderer. Use `ComarkRenderer` with the tree returned by `renderMarkdown`.
- Prefer annotating the Comark tree with heading IDs over render-time heading counters. This avoids React Strict Mode/render-order instability.
- `ComarkRenderer` wraps output in `<div class="comark-content ...">`; CSS should still match because it remains inside `.markdown-body`. Only adjust CSS if visual verification shows a structure regression.
- Search will remain imperative DOM post-processing. That is acceptable here because markdown itself is React-rendered and the hook can fully remove its own marks before reapplying them.
- Keep Mermaid id generation deterministic per render (`mermaid-0`, `mermaid-1`, ...), matching the current behavior.

Run from repo root:


Manual checks in the desktop app:

- Markdown renders without `dangerouslySetInnerHTML` for document content.
- Headings have stable IDs; duplicate headings get `-1`, `-2`, etc.
- Sidebar outline clicks still scroll to headings.
- Scroll spy updates the active heading while scrolling.
- Syntax-highlighted code blocks retain styling and copy correct plaintext.
- Copy button state changes to copied and resets.
- Mermaid diagrams render, re-render after theme/content/tab changes, and do not duplicate stale SVGs.
- Search highlights matches, next/previous navigation works, clearing search removes all marks, and changing tabs/content does not leave stale marks.
